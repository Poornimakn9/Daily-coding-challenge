**Day 16: Customer Segmentation using SQL**
Today’s task focused on analyzing customer data and segmenting users based on their spending behavior using SQL.

I practiced core SQL concepts like SELECT, WHERE, GROUP BY, ORDER BY, CASE, and HAVING to extract insights from the dataset.

**Key Highlights:**

Filtered customers based on city, age, and spending
Sorted data to identify high-value customers
Calculated total revenue and average spending
Grouped data by city to analyze regional performance
Created customer segments (Low, Medium, High) using CASE
Applied HAVING clause to filter high-performing cities

This task helped in understanding how businesses categorize customers and make data-driven decisions using SQL.
