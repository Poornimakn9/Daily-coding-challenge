# Day 23 – String-Based Complaint Analysis | Ola Case Study

## Project Summary

This Day 23 challenge demonstrates how raw textual data can be analyzed using **only string operations**.

---

##  Objective

Analyze ride complaint data without using advanced data structures or external libraries — relying strictly on **Python string manipulation techniques**.

---

##  Dataset Description

The complaint data is received as one continuous string containing:

* Ride ID
* Driver Name
* Complaint Type
* City

Each record is separated by commas.

---

##  Tasks Implemented

✔ Convert entire dataset to lowercase for uniform processing
✔ Determine total number of complaint records
✔ Identify frequency of *late arrival* issues
✔ Count complaints originating from Bangalore
✔ Extract the first driver's name from the dataset
✔ Standardize complaint labels for consistency

---

##  Skills Applied

* Text preprocessing
* Data cleaning fundamentals
* Pattern recognition using strings
* Basic data standardization techniques
* Analytical thinking without data structures

---

##  Tools & Concepts

* Python
* String Methods (`lower`, `count`, `split`, `replace`)
* Logical data parsing

---

##  Key Takeaway

Real-world data often arrives in messy formats.
This challenge highlights how meaningful insights can still be extracted using **simple yet powerful string operations**.
